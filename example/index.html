⚡ AI Workflow Patterns

Выберите правильный паттерн для вашей задачи

1. Explore → Plan → Code → Review

Для сложных фич: Понимание кода → планирование → реализация → проверка качества

Используется в: Разработка новых функций, архитектурные изменения

2. Prototype → Rewrite → Improve

Для экспериментов: Быстрая реализация → изучение → переписывание с лучшей архитектурой

Используется в: POC, исследовательские задачи, незнакомые домены

3. Build → Integrate → Delegate → Iterate

Для автоматизации: Создание инструментов → интеграция с AI → делегирование → улучшение

Используется в: Повторяющиеся задачи, улучшения CI/CD, инструменты

🆕 AI/tasks Integration

Полный lifecycle задачи: AI/tasks/$TICKET содержит все этапы от планирования до завершения

  • JIRA_TASK.md: Описание задачи
  • SUB_JIRA_TASK.md: Атомарные подзадачи с чекбоксами
  • NOTES.log: Детальный лог выполнения
  • JIRA_UPDATED_TASK.md: Финальный отчет
Слайд 5

🔄 Development Cycle - Iterative Coding

Непрерывная разработка с поддержкой AI:

# Typical development loop
while [ $task_complete = false ]; do
  1. Write code with AI assistance
  2. Run automated tests: make test
  3. Check linting: make lint
  4. AI code review: AI/code-review/HOW_TO_USE.md
  5. Iterate based on feedback
done

Ключевые практики:

  • Небольшие, тестируемые инкременты
  • Непрерывная валидация
  • Самопроверка перед человеческим ревью

🔍 Code Review Integration

Многоуровневый процесс AI-ревью

1. 👤 Self-Review

Локально

  • AI/code-review шаблоны

2. 🤖 CI Integration

Автоматически

  • Diff Critic: построчные комментарии
  • Custom AI Review: комплексный анализ

3. 👥 Team Review

Совместно

  • AI-генерированные инсайты
  • Человеческий контроль

Diff Critic Features:

  • Context providers (сейчас Git, в будущем: microservices + monoliths)
  • OpenAI анализ с кастомными промптами
  • Автоматические комментарии в GitLab MR
Слайд 7

📋 Team Guidelines & Best Practices

✅ Must Do

  • Обдуманно проверять каждую строку перед MR
  • Документировать AI-сгенерированный код (1-2 предложения)
  • Использовать самопроверку с AI перед командным ревью
  • Проверять встроенные альтернативы перед добавлением зависимостей

❌ Avoid

  • Слепое доверие AI предложениям
  • Добавление ненужных зависимостей
  • Отправка непроверенного AI-кода

🔄 Процесс:

  • Обсуждать спорные AI советы в комментариях MR
  • Делиться находками в командном чате
  • Документировать критические промпты
